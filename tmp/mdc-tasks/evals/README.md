# 独立版任务拆解评测

这个目录包含独立版 `mdc-tasks` 的评测 prompts。

## 目的

这些评测用于验证该 skill 是否真正做到：

- 基于需求文档和设计文档拆解任务，而不是脱离输入自由发挥
- 在缺少设计、设计仍不完整或存在冲突时，明确停在“还不能可靠拆解”的状态
- 在范围过大时，先切分计划边界，而不是直接写一份失真的总计划
- 输出里程碑、任务、依赖、验证方式和完成定义
- 保持任务与需求、设计之间的追溯关系
- 在结束时明确给出 `可进入实现` 或 `仍需补齐输入` 的判断

## 建议评分关注点

1. 是否真的使用需求和设计作为任务拆解依据
2. 是否在设计缺失、设计仍不足或输入冲突时停止硬拆任务
3. 是否在超大范围场景下先给出 2-3 种切分方式并推荐一份计划边界
4. 是否把任务写到足够可执行、可验证，而不是停留在大块工作项
5. 是否明确写出依赖、验证方式、完成定义和追溯关系
6. 是否遵循用户指定的路径、模板或章节骨架
