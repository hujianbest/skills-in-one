# AI辅助研发主地图设计文档

**目标：** 在不新增辅助工作表的前提下，补全 `AI辅助开发沙盘V2.xlsx` 当前主地图页，使其形成一张可用于评审和落地推进的 AI 辅助研发作业地图。

**范围：**
- 保留用户当前主地图页已有的领域、事项、活动分解和痛点表述风格。
- 重点补全以下列：`执行方式`、`AI完成活动中的什么任务`、`AI解决措施`、`人在活动中完成什么任务`、`AI执行的潜在问题`、`优先级`、`AI解决了哪些痛点`。
- 不新增辅助工作表，不改变用户要求的主交付边界。

**设计原则：**
- 以软件开发全作业流为主线，内容覆盖设计、开发、测试、运维、质量治理等关键活动。
- `AI解决措施` 必须写到可实施层面，明确工具、Agent、触发时机、集成系统和人工关卡。
- 保持“人机协同”视角，不把关键责任完全外包给 AI。
- 所有高风险场景均保留人工确认、审批或定版环节。

**输出结构：**
1. 每个事项明确 AI 与人的分工边界。
2. 每个事项明确推荐执行方式：`AI独立解决`、`人和AI合作`、`人独立解决`。
3. 每个事项将业务痛点映射到具体 AI 作业动作和落地收益。

**推荐落地写法：**
- 编码场景：在 IDE 中启用 `opencode` 的 TDD 技能，接入仓库级 RAG、规范模板和脚手架。
- 代码检视场景：在 Git MR/PR 门禁中部署 AI Review Agent，自动读取 Diff、规则库和历史缺陷。
- 质量门禁场景：在 CI/CD 中部署 AI Quality Gate Agent，与静态扫描、测试覆盖、兼容性检查串联。
- 问题定位场景：在日志、Trace、监控平台部署定位 Agent，自动聚合证据并回写工单。

**验收标准：**
- 主地图页关键空白列被完整补齐。
- 新增内容具体、可执行，而非停留在“AI辅助分析”“AI提升效率”等抽象描述。
- 至少覆盖用户已输入的主要研发活动，并体现企业级工具链和流程打通思路。
