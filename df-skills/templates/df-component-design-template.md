# df Component Implementation Design Template

> **占位说明**
>
> 这个文件是 `df-component-design` 写作组件实现设计时使用的团队模板。
>
> df 硬约定：**组件实现设计必须符合团队既有模板（Template-Constrained Design）**。当前模板内容**留空**，由团队在落地 df 时手动补齐。
>
> 在团队补齐前，`df-component-design` 仍可执行，但写出的设计文件需由模块架构师手工把模板章节补全后再交评审。
>
> 推荐补齐方向（最小章节集，详见 `docs/df-shared-conventions.md`）：
>
> - 组件职责与非职责
> - SOA 服务 / 接口
> - 依赖组件
> - 数据模型与状态机
> - 并发 / 实时性 / 资源生命周期
> - 错误处理与降级策略
> - 配置项与编译条件
> - 对 AR 实现设计的约束
