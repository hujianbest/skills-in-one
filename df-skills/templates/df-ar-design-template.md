# df AR Implementation Design Template

> **占位说明**
>
> 这个文件是 `df-ar-design` 写作 AR 实现设计时使用的团队模板。
>
> df 硬约定：**AR 实现设计必须符合团队既有模板**，并且 **测试设计是 AR 实现设计的章节**（不作为独立 `test-design.md` 文件）。当前模板内容**留空**，由团队在落地 df 时手动补齐。
>
> 在团队补齐前，`df-ar-design` 仍可执行，但写出的 AR 设计文件需由开发负责人 / 模块架构师手工把模板章节补全后再交评审。
>
> 推荐补齐方向（最小章节集，详见 `docs/df-shared-conventions.md`）：
>
> - AR ID、SR link、所属组件
> - 设计目标与范围
> - 受影响文件 / 模块 / 接口
> - 数据结构与控制流
> - C / C++ 实现策略（命名、错误处理、并发、内存、实时性）
> - 与组件实现设计的一致性说明
> - 测试设计章节（用例列表、覆盖目标、预期 I/O、mock / stub / 仿真说明、RED / GREEN 证据要求）
